# AGENTS.md Template

Use this shape for proposed root or nested agent guidance.

```markdown
# Repository Guidelines

## Scope

These instructions apply to `<scope>`.

## Project Map

- `<path>`: `<purpose>`

## Build, Test, and Validation

- `<command>`: `<when to use it>`

## Coding Conventions

- Observed: `<fact with cited source path>`
- Recommendation: `<specific agent instruction>`

## Testing Expectations

- `<test command or expectation>`

## Security and Data Handling

- Do not read, print, or commit secret values.
- Treat `<path>` as sensitive/risky because `<reason>`.

## Agent Workflow

- Check existing user changes before editing.
- Keep source edits scoped to the requested task.
- Update docs or compatibility guidance when behavior changes.
```
```

Root guidance should cover repo-wide defaults. Nested guidance should exist only where local behavior differs materially from the root.
